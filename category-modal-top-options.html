Implementation

Option 3 — React Native Code

useSafeAreaInsets() from react-native-safe-area-context returns the device's inset values (including Dynamic Island / notch height). Pass insets.top as a style prop to the sidePanel.

// CategoryModal.tsx import { useSafeAreaInsets } from 'react-native-safe-area-context'; export function CategoryModal({ ... }) { const insets = useSafeAreaInsets(); return ( <Modal ...> <View style={styles.modalContainer}> <TouchableWithoutFeedback onPress={onClose}> <View style={styles.backdrop} /> </TouchableWithoutFeedback> <Animated.View style={[ styles.sidePanel, { marginTop: insets.top + 8 }, // ← safe area + 8pt gap ]} > ... </Animated.View> </View> </Modal> ); } // styles.ts — add borderTopLeftRadius sidePanel: { width: SCREEN_WIDTH * 0.75, maxWidth: 500, backgroundColor: colors.surface, borderTopLeftRadius: 16, // ← rounded top-left corner ...boxShadow(0, 12, 'rgba(0,0,0,0.25)', -4), },

1Current (Bug)

  • Panel starts at y=0, behind the Dynamic Island
  • Header title and subtitle invisible to the user
  • Close button is also behind the notch
  • Occurs on all devices with a notch/Dynamic Island

2Safe Area Flush

  • Panel top = insets.top exactly
  • Header fully visible, no gap
  • Square top corner — feels like a hard cut
  • Simplest code change, minimal visual impact

3Safe Area + Rounded (Recommended)

  • Panel top = insets.top + 8
  • 16 pt borderTopLeftRadius on sidePanel
  • Backdrop still covers full screen including gap
  • Best balance of correctness and polish

4Sheet Style

  • Panel top = insets.top + 20
  • 20 pt radius + drag handle at panel top
  • Largest visual change, most native iOS feel
  • Implies future gesture-dismiss support
